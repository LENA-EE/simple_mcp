# Perl Code Review Skill (для LLM)

## Когда использовать

Когда нужно проанализировать Perl файл на соответствие Perl Best Practices.

## Шаги для LLM:

### Шаг 1: Получи код файла

- Убедись что у тебя ПОЛНЫЙ код

### Шаг 2: Вызови MCP tool

Используй `perlcritic_analyze` tool:

```
- name: perlcritic_analyze
- arguments:
  code: <ПОЛНЫЙ код файла>
  severity: 1
```

### Шаг 3: Проверь результат

- MCP вернёт список проблем с номерами строк

### Шаг 4: Форматируй ответ

**Обязательно покажи:**

1. ОБЩЕЕ количество проблем
2. КАЖДУЮ проблему с номером строки
3. Полиси (имя правила)
4. Рекомендацию по исправлению опиши сам, используя best practice к данной проблеме

**Формат вывода:**

=== НАЙДЕНО N ПРОБЛЕМ ===

--- Severity 1 / САМЫЙ СТРОГИЙ (N шт.) ---
1. Строка N: <проблема>
   Policy: <полиси>
   Рекомендация: <что делать>

2. Строка N: <проблема>
   ...

--- Severity 2 / СТРОГИЙ (N шт.) ---
1. Строка N: <проблема>
   ...

... и так для всех уровней 1-5

**Группировка ОБЯЗАТЕЛЬНА:**
- Сначала Severity 1 (все нарушения)
- Потом Severity 2, 3, 4, 5

---

## Важные правила для LLM:

1. НЕ обрезай код перед отправкой в MCP!
2. НЕ суммируй — показывай ВЫХОД MCP построчно!
   - КАЖДУЮ проблему с отдельной строки
   - С номером строки
   - С Policy
   - С рекомендацией
3. ВСЕГДА указывай номер строки
4. severity=1 обязательно (иначе много пропустит)
5. не обманывай и работай честно
